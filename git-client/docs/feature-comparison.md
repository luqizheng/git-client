# Git客户端功能对比文档

对比GitKraken，梳理已实现和未实现的功能。

## 已实现功能 ✅

### 核心Git操作
- [x] 打开/克隆仓库
- [x] 提交（Commit）
- [x] 暂存/取消暂存（Stage/Unstage）
- [x] 推送/拉取/获取（Push/Pull/Fetch）
- [x] 分支管理（创建、切换、删除、重命名）
- [x] 标签管理（创建、列表、删除）
- [x] 储藏（Stash）
- [x] 变基（Rebase）
- [x] 合并（Merge）与冲突解决
- [x] Cherry-pick
- [x] 重置（Reset - Soft/Mixed/Hard）
- [x] 撤销提交（Revert）

### 视图与界面
- [x] 提交历史列表（虚拟滚动）
- [x] 提交图谱（Commit Graph）基础版
- [x] 分支树形展示
- [x] 文件差异查看（Monaco Editor）
- [x] 分支比较
- [x] 三路合并冲突解决
- [x] 深色/浅色主题
- [x] 国际化（中/英）

### 高级功能
- [x] 子模块（Submodule）管理
- [x] 工作树（Worktree）管理
- [x] GitFlow工作流
- [x] Git钩子（Hooks）管理
- [x] 高级搜索
- [x] SSH配置
- [x] 多仓库标签页

### 交互体验
- [x] 右键上下文菜单
- [x] 键盘快捷键
- [x] 文件变更实时监听
- [x] 提交详情面板

## 缺失功能 ❌

### 1. 代码追溯与审查
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **Git Blame** | 高 | 行级代码作者追溯，查看每行代码最后修改者 |
| **代码注释/讨论** | 中 | 对提交或代码行添加评论（类似GitKraken的Code Review） |

### 2. 提交图谱增强
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **交互式Rebase** | 高 | 可视化拖拽调整提交顺序、压缩、删除提交 |
| **提交筛选器** | 中 | 按作者、日期、文件路径筛选提交 |
| **图谱分支高亮** | 中 | 鼠标悬停高亮整条分支线 |
| **迷你图谱导航** | 低 | 右侧缩略图快速导航 |

### 3. 工作区与文件管理
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **Patch应用** | 中 | 应用/导出Patch文件 |
| **文件历史** | 中 | 查看单个文件的修改历史 |
| **忽略文件模板** | 低 | 快速添加.gitignore规则 |
| **大文件提示** | 低 | 提交前警告大文件 |

### 4. 团队协作与集成
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **GitHub/GitLab集成** | 中 | 查看PR/MR、创建PR、查看CI状态 |
| **提交签名(GPG)** | 低 | GPG签名验证与提交 |
| **提交模板** | 低 | 自定义提交信息模板 |

### 5. 生产力工具
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **Quick Open / Fuzzy Finder** | 高 | 快速打开文件或分支 |
| **命令面板** | 中 | 类似VS Code的命令面板 |
| **工作区(Workspaces)** | 中 | 保存多仓库组合为工作区 |
| **提交书签** | 低 | 标记重要提交 |

### 6. 高级Git功能
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **Git Bisect** | 低 | 二分查找引入bug的提交 |
| **Reflog查看** | 中 | 查看引用日志，恢复误删分支 |
| **Bundle操作** | 低 | 打包/解包仓库 |
| **Subtree** | 低 | 子树合并管理 |

### 7. 差异与合并增强
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **图像差异对比** | 低 | 图片文件的视觉对比 |
| **二进制文件处理** | 低 | 二进制文件差异提示 |
| **合并工具配置** | 低 | 配置外部合并工具 |

### 8. 可视化与报告
| 功能 | 优先级 | 说明 |
|------|--------|------|
| **贡献统计** | 低 | 代码贡献者统计图表 |
| **代码行数统计** | 低 | 仓库代码量统计 |

## 功能实现优先级建议

### P0 - 核心缺失（尽快实现）
1. **Git Blame** - 代码审查必备
2. **交互式Rebase** - 高级Git用户高频使用
3. **Quick Open** - 提升操作效率

### P1 - 重要功能（后续迭代）
4. Reflog查看
5. 文件历史
6. Patch应用
7. 提交筛选器
8. 命令面板

### P2 - 增强体验（长期规划）
9. GitHub/GitLab集成
10. 工作区管理
11. 图谱分支高亮
12. 代码注释/讨论

### P3 - 锦上添花
13. 贡献统计
14. 图像差异
15. Git Bisect
16. 其他高级功能

---

## 备注

- 当前项目已实现GitKraken约 **70%** 的核心功能
- 主要差距在：代码追溯(Blame)、交互式Rebase、Quick Open、外部集成
- 建议优先实现Blame和交互式Rebase，这两项是专业Git用户的刚需
