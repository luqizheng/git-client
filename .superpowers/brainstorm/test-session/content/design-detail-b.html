🎯 核心交互逻辑

默认状态（无选中）

  • 左侧：完整导航栏 (180px)
  • 中央：Commit Graph 全宽显示
  • 底部：StageArea + CommitEditor 固定 (200px)
  • 右侧：隐藏

点击 Commit 后

  • 右侧面板从边缘滑出 (320px)
  • 中央区域自适应缩小
  • 显示 commit 详情 + 文件列表
  • 可拖拽调整右侧面板宽度

点击文件后

  • Diff 内容替换中央区域内容
  • 保留左右面板不变
  • 最大化 diff 查看面积
  • 支持 Monaco Editor 三路对比

返回 Graph 视图

  • 点击 "Back to Graph" 按钮
  • 或按 Escape 键返回
  • 右侧面板保持打开状态

🔧 技术实现要点

1. 可拖拽分栏： 使用自定义 ResizeHandle 组件，监听 mousedown/mousemove/mouseup 事件，动态更新 store 中的宽度值。

2. 右侧面板动画： CSS transition: width 0.2s ease, opacity 0.2s ease；使用 v-show 控制显隐避免销毁组件。

3. 视图切换： 中央区域使用动态组件 <component :is="currentView"> 切换 GraphView ↔ DiffView；通过 Pinia store 管理当前视图状态。

4. NaiveUI Scrollbar： 所有滚动区域统一使用 <n-scrollbar> 组件，配置 themeOverrides 自定义深色样式。

5. 响应式断点： 窗口 < 1024px 时自动隐藏右侧面板，< 768px 时隐藏左侧导航（仅图标模式）。
