方案 A: 经典三栏布局

优点：

  • 完全还原 GitKraken 经典体验
  • 信息密度高，一屏览全局
  • 符合用户已有认知习惯

缺点：

  • 小屏幕下空间紧张

适用： 桌面端主力使用场景

方案 B: 宽屏优化 + 隐藏面板
⌘ Toolbar
📂 Nav
🌿 Branches
📡 Remotes
📊 Commit Graph (Maximized)
● feat: add new feature
● fix: resolve bug
● docs: update readme
→ 点击 commit 时右侧面板滑出 →

方案 B: 宽屏优先 + 按需展开

优点：

  • 主工作区最大化，专注代码审查
  • 右侧面板按需显示，减少干扰
  • 适合大屏显示器 (≥1920px)

缺点：

  • 频繁查看详情时需要反复操作
  • 不能同时看到 graph 和 details

适用： 大屏幕、重度 diff 使用场景

方案 C: 紧凑图标模式

优点：

  • 左侧极简，为主区域腾出空间
  • 图标导航现代感强
  • 适合笔记本/小屏设备

缺点：

  • 导航项多时需要 hover/二次点击
  • 信息层级不够直观

适用： 笔记本、便携优先场景
