# 待规划 - UI 已有但后端缺失的功能

以下功能在 GitKraken UI 还原中需要，但当前 Rust 后端尚无对应 Tauri Command。

---

## 1. Undo / Redo（撤销/重做）
- **UI 位置**: Toolbar 左侧 Undo/Redo 按钮
- **缺失内容**:
  - `undo` 命令
  - `redo` 命令
  - 操作历史记录管理（AppState 中维护 undo stack / redo stack）
- **涉及操作**: branch switch, branch delete, commit, merge, rebase, reset, stash 等大部分写操作
- **优先级**: 中（UI 按钮可先灰显）

## 2. Pull 高级选项
- **UI 位置**: Toolbar Pull 按钮下拉菜单
- **缺失内容**:
  - `pull_ff_only` — `git pull --ff-only`
  - `pull_rebase` — `git pull --rebase`
  - 当前 `pull` 命令仅支持默认 merge pull
- **已有**: `fetch`（可复用为 Fetch All）、`pull`（默认 ff-if-possible）
- **优先级**: 高

## 3. Stash Apply（应用但不删除）
- **UI 位置**: 左侧 Stashes 分区右键菜单 → Apply Stash
- **缺失内容**:
  - `stash_apply` 命令（apply stash 但保留 stash 条目）
- **已有**: `stash_pop`（apply + drop）
- **优先级**: 高

## 4. Stash Drop / Delete
- **UI 位置**: 左侧 Stashes 分区右键菜单 → Delete Stash
- **缺失内容**:
  - `stash_drop` 命令（删除指定 stash 条目）
- **优先级**: 高

## 5. Branch Rename（分支重命名）
- **UI 位置**: 左侧 Local 分区右键菜单 → Rename
- **缺失内容**:
  - `rename_branch` 命令
- **优先级**: 中

## 6. Ahead / Behind 计数
- **UI 位置**: Status Bar 显示 "2 ahead, 1 behind"
- **缺失内容**:
  - `get_ahead_behind` 命令（对比本地分支与上游分支的差异计数）
- **优先级**: 高

## 7. Tag Push（推送标签）
- **UI 位置**: 左侧 Tags 分区右键菜单 → Push Tag
- **缺失内容**:
  - `push_tag` 命令（推送指定标签到远程）
- **已有**: `push`（仅推送分支）
- **优先级**: 中

## 8. Merge Branch（合并分支）
- **UI 位置**: 左侧分支右键菜单 → Merge into current
- **缺失内容**:
  - `merge_branch` 命令（将指定分支合并到当前分支）
- **已有**: `cherry_pick`、`revert_commit`、`rebase_branch`
- **优先级**: 高

## 9. Submodule Remove / Deinit
- **UI 位置**: 左侧 Submodules 分区右键菜单
- **缺失内容**:
  - `deinit_submodule` 命令（移除子模块）
- **已有**: `list_submodules`、`init_submodule`、`update_submodule`
- **优先级**: 低

## 10. Fetch All（获取所有远程）
- **UI 位置**: Toolbar Pull 下拉 → Fetch All
- **缺失内容**:
  - `fetch_all` 命令（从所有远程 fetch，而非仅指定一个远程）
- **已有**: `fetch`（仅支持指定单个远程名）
- **优先级**: 高

## 11. Remote Branch Listing（远程分支列表）
- **UI 位置**: 左侧 Remote 分区展开远程名显示远程分支
- **缺失内容**:
  - `list_remote_branches` 命令（列出指定远程的所有分支）
  - 或在 `list_branches` 返回中区分 local/remote
- **已有**: `list_branches`（需确认是否已返回远程分支）
- **优先级**: 高

---

## 实施建议

- **先 UI 后后端**: UI 上的这些按钮/菜单先渲染，点击时给出 "功能开发中" 提示（toast）
- **分批实现后端**: 按优先级排序，第一批实现 #2 #3 #4 #6 #8 #10 #11
- **Undo/Redo 复杂度高**: 建议作为独立迭代，不影响其他功能
